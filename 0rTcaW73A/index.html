[Go] 2.Map

Posted on 2021-02-02，4 min read

定义

是无序的键值对，类似python的字典。

语法

使用关键字map来声明：
map[KeyType]ValueType
注意：
1.必须指定key,value的类型，插入的记录类型必须匹配。
2.key具有唯一性，不能重复。
3.KeyType可以是基础数据类型（例如bool,数字类型、字符串），不能是数组、切片、map,它的取值必须是能够使用 == 进行比较。
4.ValueType可以是任意类型。
5.无序性
6.线程不安全，一个gotuntine在对map进行写的时候，另外的goroutine不能进行读和写操作，Go1.6版本以后会抛出runtime错误信息。

声明和初始化

1.使用var声明

var cMap map[string]int  //只定义（声明），此时cMap为nil
fmt.Println(cMap == nil)
cMap["北京"] = 1 //报错，因为cMap为nil
//在进行赋值之前，需要先用make进行初始化
cMap = make(map[string]int)
cMap["北京"] = 1 //此时就不会报错了。

2.使用make进行短声明

cMap := make(map[string]int) //进行来初始化
cMap["北京"] = 1

//指定初始容量
cMap = mkae(map[string]int,100)
cMap["北京"] = 1

说明：在使用make初始化map的时候，可以指定初始化容量，这个能在预估map的key数量的情况下，减少动态分配的次数，从而提升性能。
3.简短声明

cMap := map[string]int{"北京":1}

4.var、make结合
var cMap = make(map[string]int)
cMap["北京"] = 1
注意：
从上面4种方法看下来，如果是直接用var,且声明的时候没有进行初始化的（没有make或者赋初始值），map的值是nil，要赋值之前需要先make初始化才行。

基本操作

cMap := map[string]int{}
cMap["北京"] = 1 //写

code := cMap["北京"]  //读
fmt.println(code)

code = cMap["广州"] //读不存在的key
fmt.Println(code)

code,ok = cMap["广州"] //检查key是否存在
if ok {
    fmt.Println(code)
}else{
    fmt.Println("key not exist")
}

delete(cMap,"北京") //删除指定key
fmt.Println("北京")

循环和无序性

cMap := map[string]int{"北京":1,"上海":2,"广州":3,"深圳":4}

for city,code := range cMap {  //因为无序性，所以每次迭代的结果的顺序都不同。
    fmt.Printf("%s:%d",city,code)
    fmt.Println() 
}

线程不安全

cMap := make(map[string]int)
var wg sync.WaitGroup
wg.Add(2)

go func(){
    cMap["北京"] =1
    wg.Done()
}()

go func(){
    cMap["上海"] = 2
    wg.Done()
}()

wg.Wait()

在Go1.6之后的版本，多次运行此段代码，将会遇到如下错误信息：

fatal error: concurrent map writes

goroutine x [running]
runtime.throw(0x10c64b4,0x15)
....

解决方法：
1.对读写操作加锁
2.使用security map，例如 sync.map

map嵌套

provinces := make（map[string]map[string]int)  //map中key的value也是map

provinces["北京"] = map[string]int{
    "东城区":1,
    "西城区":2,
    "朝阳区":3,
    "海淀区":4,
}
fmt.Println(provinces["北京"])
