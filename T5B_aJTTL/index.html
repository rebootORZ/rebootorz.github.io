FRP多层穿透实战

Posted on 2021-01-11，6 min read

FRP介绍

frp 是一个高性能的反向代理应用，可以使目标对外开放端口，支持 tcp, http, https 等协议类型，并且 web 服务支持根据域名进行路由转发，从而在红队行动时实现内网穿透。

项目地址

https://github.com/fatedier/frp/blob/master/README_zh.md

实验内容

完成两种穿透：
A-B-DMZ-内网A
A-B-DMZ-内网A-内网B

前期准备

Frp分为客户端和服务端，且有windows和linux版本，服务端开放一个端口，供客户端连接，客户端开放端口，供攻击者通过服务端来访问客户端。所以我们需要对其配置文件进行配置。
** 服务端通用配置模板： **

#通用配置段

[common]

#frp服务端监听

bind_addr = 0.0.0.0

#frp服务器监听端口 [实战中根据实际情况选择可通信的端口]

bind_port = 7000

 

#服务端Web控制面板登录端口 [通过控制面板，可以实时了解到数据收发情况。一般用不到]

dashboard_port = 6609

#服务端Web控制面板用户名与密码 [强口令]

dashboard_user = rebootORZ

dashboard_pwd = Admin@123

 

#日志输出位置

log_file = ./frps.log

#日志记录等级，有trace、debug、info、warn、error,通常情况下为info

log_level = info

#日志保留时间

log_max_days = 3

 

#验证凭据，服务端和客户端的凭据必须一样才能连接

auth_token = Admin@auth

#启用特权模式，从v0.10.0版本开始默认启用特权模式 [特权模式下，客户端更改配置无需更新服务端]

privilege_mode = true

#特权模式Token [强口令，建议随机生成]

privilege_token = Admin@privilege

#特权模式允许分配的端口 [避免端口被滥用]

privilege_allow_ports = 4000-50000

 

#心跳检测超时时长

heartbeat_timeout = 30

 

#每个代理可以设置的连接池上限

max_pool_count = 20

 

#口令认证超时时间，一般不用改

authentication_timeout = 900

 

#指定子域名，后续将全部用域名的形式进行访问 [特权模式需下将 *.xxxx.online 解析到外网VPS上，即域名泛解析]

#subdomain_host = xxxx.online

** 客户端通用配置模板： **

#通用配置段

[common]

#frp服务端IP或域名 [实战中一般都会直接用域名]

server_addr = XXX.XXX.XXX.XX

#frp服务器端口

server_port = 7000

 

#授权token，此处必须与服务端保持一致，否则无法建立连接

auth_token = Admin@auth

#启用特权模式 [特权模式下服务端无需配置]

privilege_mode = true

#特权模式 token,同样要与服务端完全保持一致

privilege_token = Admin@privilege

 

#心跳检查间隔与超时时间

heartbeat_interval = 10

heartbeat_timeout = 30

 

#启用加密 [通信内容加密传输，有效防止流量被拦截]

use_encryption = true

#启用压缩 [传输内容进行压缩，有效减小传输的网络流量，加快流量转发速度，但会额外消耗一些CPU资源]

use_compression = true

 

#连接数量

pool_count = 20

remote_port = 9066

#使用插件socks5代理

plugin = socks5

#socks5连接口令 [根据实际情况进行配置]

#plugin_user = rebootORZ

总结

其实我们打通服务器C的时候，由于C和G都是双网卡的跨网段机器，通过C去打内网A,再通过G去打内网B，完全可行了，但是在实际场景中，特别是红队场景，往往是需要多个人同时参与到攻击中，所以需要利用FRP来创建这个隧道，使得更多的人可以进到内网。
