[Go]普通函数调用与闭包的区别

2021-01-26 2 min read # Golang 
package main


import (
　　"fmt"
)

 func main() {
　　 a := []int{1, 2, 3}
 　　for _, i := range a {
　　 　　fmt.Println(i)
　　　　 defer p(i)
　　 }
 }
 
 func p(i int) {
　　fmt.Println(i)
 }

运行这段程序，输出结果为


1

2

3

3

2

1

这里就是普通的函数调用，每次调用func p时，完成 i 的值复制，然后打印，此时 i 值复制了3次，分别是1，2，3。由于defer是后进先出，所以执行变成3，2，1。

下面我我们用闭包改写下程序：


package main

import (
　　"fmt"
)

func main() {
　　a := []int{1, 2, 3}
　　for _, i := range a {
　　　　fmt.Println(i)
　　　　defer func() {
  　　　　 fmt.Println(i)
　　　　}()
    }
}

运行这段程序，输出结果为


1

2

3

3

3

3

在闭包函数里，这个i是闭包函数外面的参数（即不是闭包函数中的参数），所在当在fmt.Println(i)中进行引用的时候，因为闭包函数是“地址引用”，而非“值引用”，所以前面正常输出是1、2、3，而到了defer时，按照原本的“后入先出”进行输出的话，但由于到了这里的时候i的地址上是3，虽然参数i的地址没变，但是值变了，所以在Println(i)的时候输出都是3，而不是3、2、1。
